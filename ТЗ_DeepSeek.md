# Функциональные требования (Wiki-Explorer-DS CLI)

## Общее поведение CLI
- Название утилиты: `wiki-explorer-ds`
- Все команды принимают глобальные опции:
  - `--lang` (`-l`) — язык Wikipedia (по умолчанию `ru` или `en`)
  - `--verbose` — вывод отладочных сообщений (запросы к API, предупреждения)
- Вывод — в читаемом табличном виде (используется `rich` или просто отформатированные строки). Дополнительных форматов (`json`, `plain`) нет.
- Ошибки: сообщение с пояснением, код завершения `≠ 0`.
- Кэширование (опционально): реализуется через SQLite, время жизни кэша — одна сессия.

---

## Команда 1: `info`
**Назначение:** метаинформация о статье.  
**Аргументы:**  
- `title` (обязательный)

**Опции:**  
- `--show-categories` — вывести список категорий (первые 10)  
- `--show-image-url` — вывести URL главного изображения (если есть)

**Выводимые поля:**  
- Заголовок  
- Размер в байтах  
- Количество внутренних ссылок  
- Дата создания (самая ранняя ревизия)  
- Дата последней правки  
- Последний редактор  
- Категории (если флаг)  
- URL изображения (если флаг)

**Пример:**  
```bash
wiki-explorer-ds info "Python" --lang en --show-categories
```

---

## Команда 2: `links`
**Назначение:** список внутренних ссылок (куда ссылается статья).  
**Аргументы:**  
- `title`

**Опции:**  
- `--limit N` (по умолчанию 50, макс. 500)  
- `--offset N` (пагинация, по умолчанию 0)  
- `--search SUBSTRING` — фильтр по названиям ссылок (регистронезависимо)

**Вывод:** таблица `# | Название ссылки`.  

**Пример:**  
```bash
wiki-explorer-ds links "Artificial intelligence" --limit 30 --search "learning"
```

---

## Команда 3: `graph`
**Назначение:** построение графа ссылок и сохранение в PNG.  
**Аргументы:**  
- `title`

**Опции:**  
- `--output PATH` (обязательный, путь к файлу `.png`)  
- `--depth {1,2}` — глубина графа:  
  - `1` — только исходная статья и её прямые ссылки  
  - `2` — исходная статья → её ссылки → ссылки этих ссылок (прямые второго уровня)  
- `--max-links N` — ограничить количество целевых статей (чтобы граф не был перегружен)

**Результат:**  
Сохраняется PNG-файл. В консоль выводится сообщение `Граф сохранён в <путь>`.  
Никакого текстового вывода рёбер или интерактивного просмотра.

**Пример:**  
```bash
wiki-explorer-ds graph "Machine learning" --output ml_graph.png --depth 2 --max-links 20
```

---

## Команда 4: `categories`
**Назначение:** показать категории статьи.  
**Аргументы:**  
- `title`

**Опции:**  
- `--tree` — вывести плоский список с указанием родительской категории (без рекурсии, только один уровень родителей). Реализуется через отдельный запрос к API для каждой категории (допускается медленная работа).

**Вывод:**  
- Без `--tree`: плоский список категорий.  
- С `--tree`: таблица `Категория | Родительская категория`.

**Пример:**  
```bash
wiki-explorer-ds categories "France" --tree
```

---

## Команда 5: `pageviews`
**Назначение:** статистика просмотров за период.  
**Аргументы:**  
- `title`

**Опции:**  
- `--days N` (по умолчанию 30, максимум 90) — количество последних дней (включая сегодня, если данные есть)  
- `--chart ascii|png` — если `ascii`: столбчатая диаграмма в консоли, если `png`: сохранить файл `pageviews_<title>.png` в текущую папку (или указать через `--output`)  
- `--output PATH` — для PNG (необязательно)

**Вывод:**  
Таблица `Дата | Просмотры`, а также сумма, среднее, максимум, минимум.  
При `--chart ascii` — график из символов.

**Пример:**  
```bash
wiki-explorer-ds pageviews "Python" --days 60 --chart ascii
```

---

## Команда 6: `random`
**Назначение:** случайная статья с фильтрацией.  
**Аргументы:** нет.

**Опции:**  
- `--category CATEGORY` — ограничиться статьями из указанной категории (реализуется через несколько попыток или через `generator=categorymembers`, допустима упрощённая реализация без категории)  
- `--min-words N` — минимальное количество слов в статье (проверяется через `prop=extracts`)  
- `--with-image` — требовать наличие хотя бы одного изображения (`prop=images`)  
- `--max-attempts N` (по умолчанию 10) — сколько попыток подобрать статью, удовлетворяющую фильтрам

**Вывод:**  
Название статьи, её первое предложение (краткое описание), URL статьи, отметка о выполненных фильтрах.

**Пример:**  
```bash
wiki-explorer-ds random --min-words 500 --with-image --category "Science"
```

---

## Команда 7: `search`
**Назначение:** поиск статей по ключевым словам.  
**Аргументы:**  
- `query` (поисковый запрос)

**Опции:**  
- `--limit N` (по умолчанию 10, максимум 100)  
- `--sort relevance|last_edit` — сортировка результатов  
- `--namespace` (по умолчанию 0 — основное пространство)

**Вывод:** таблица `Заголовок | Краткое описание (snippet) | Дата последней правки`.

**Пример:**  
```bash
wiki-explorer-ds search "neural network" --limit 15 --sort last_edit
```

---

## Команда 8: `images`
**Назначение:** список изображений и выборочное скачивание.  
**Аргументы:**  
- `title`

**Опции:**  
- `--download` — скачать все изображения (в папку по умолчанию или указанную в `--output`)  
- `--index N` — скачать только изображение под номером N (можно несколько через запятую: `--index 1,3,5`). Если не указано и нет `--download` — просто показать список.  
- `--output DIR` — папка для сохранения (по умолчанию `./wiki_images/`)

**Вывод без скачивания:** таблица `# | Название файла | URL оригинала | Размер (если доступен)`.  
**При скачивании:** прогресс и сообщение о сохранении.

**Пример:**  
```bash
wiki-explorer-ds images "Sunflower" --download --index 1,2 --output ~/pics/
```

## Дополнительные нефункциональные требования (кратко)
- Конфигурационный файл (опционально): `~/.wiki-explorer-ds/config.json` — можно хранить язык по умолчанию, папку для скачиваний.
- Обработка пагинации для `links`, `search`, `images` (через `--limit`/`--offset`).
- При недоступности API или отсутствии статьи — понятное сообщение об ошибке.
- Код должен соответствовать PEP 8, проект организован в виде Python-пакета.

## 3. Обработка ошибок

**Типы обрабатываемых ошибок**:
- `NetworkError` — нет соединения, таймаут, DNS ошибка.
- `ApiError` — API вернул ошибку (статус не 200, или MediaWiki error).
- `NotFoundError` — статья не найдена (или пустой ответ).
- `InvalidArgumentError` — пользователь ввёл неверный лимит, отрицательное количество дней и т.д.
- `ImageDownloadError` — не удалось скачать изображение.
- `GraphError` — ошибка при построении графа (нет библиотек, невалидные данные).

**Правила**:
- Каждая команда оборачивает вызовы API в `try-except`.
- При ошибке выводится сообщение красным цветом (через `rich.print("[red]...[/red]")`) и программа завершается с `sys.exit(1)`.
- В режиме `--verbose` выводится traceback.
- Не допускается падение программы с необработанным исключением.

**Пример обработки**:
```python
try:
    data = fetch_article(title)
except requests.ConnectionError:
    error("Нет соединения с Wikipedia API")
    sys.exit(1)
except ApiError as e:
    error(f"API вернул ошибку: {e.code} - {e.message}")
    sys.exit(1)
except NotFoundError:
    error(f"Статья '{title}' не найдена")
    sys.exit(1)
```

## 4. REST API endpoints (используемые)

**A. MediaWiki Action API** (базовый URL зависит от языка)  
Формат: `https://{lang}.wikipedia.org/w/api.php`

Общие параметры для всех запросов:
- `action=query`
- `format=json`
- `origin=*`
- `redirects=1`

**Endpoints по командам**:

1. **info**  
   `prop=info|revisions|categories|images`  
   `titles={title}`  
   `inprop=url|size`  
   `rvprop=timestamp|user|size`  
   `cllimit=10`

2. **links**  
   `prop=links`  
   `pllimit={limit}`  
   `ploffset={offset}`  
   `plnamespace=0`

3. **graph**  
   `prop=links` (для получения прямых ссылок)  
   При `depth=2` — дополнительный batch-запрос `titles={link1}|{link2}|...` с `prop=links`

4. **categories**  
   `prop=categories`  
   `cllimit=500`  
   Для `--tree`: дополнительный запрос `prop=categories` для каждой категории.

5. **random**  
   `list=random`  
   `rnnamespace=0`  
   `rnlimit=1` (или больше для фильтрации)

6. **search**  
   `list=search`  
   `srsearch={query}`  
   `srlimit={limit}`  
   `srsort={relevance|last_edit}`

7. **images**  
   `prop=images|imageinfo`  
   `iilimit=50`  
   `imlimit=50`

**B. Pageviews API**  
Базовый URL: `https://wikimedia.org/api/rest_v1/metrics/pageviews/per-article/{project}/{access}/{agent}/{article}/{granularity}/{start}/{end}`

Параметры:
- `project` = `{lang}.wikipedia`
- `access` = `all-access`
- `agent` = `user`
- `granularity` = `daily`
- `start/end` = `YYYYMMDDHHMMSS` (вычисляется из `--days`)

## 2. Тестирование

**Обязательные требования**:
- Использовать `pytest` (или `unittest`) для написания тестов.
- Покрытие тестами должно быть не менее 60% (основные команды и API-вызовы).
- Тесты должны быть изолированы от реальной сети — использовать `unittest.mock` или `responses` для подмены HTTP-запросов.
- Для каждой команды минимум 2 теста: успешный сценарий (мок ответа) и сценарий с ошибкой (статья не найдена, ошибка сети).

**Структура тестов**:
```
tests/
├── test_api_mediawiki.py    # тесты для функций MediaWiki API (моки)
├── test_api_pageviews.py    # тесты для Pageviews API
├── test_commands.py         # интеграционные тесты команд через CliRunner (click)
└── fixtures/                # JSON-файлы с мок-ответами
```

**Пример теста для команды `info`** (псевдокод):
```python
def test_info_success(mock_requests):
    mock_requests.get(..., json=fixture_info)
    result = CliRunner().invoke(cli, ["info", "Python"])
    assert result.exit_code == 0
    assert "Python" in result.output
```

**Важно**: при сдаче проекта тесты должны запускаться без ошибок командой `pytest`.

## 3. Конфигурация

**Поддержка конфигурационного файла** (опционально, но рекомендуется):
- Файл: `~/.wiki-explorer-ds/config.json` или `config.yaml`.
- Переменные окружения имеют приоритет над файлом, аргументы CLI — над всем.

**Параметры конфигурации**:
- `default_lang` — язык Wikipedia по умолчанию (например, `"en"` или `"ru"`).
- `image_dir` — папка для скачивания изображений по умолчанию (например, `"./wiki_images"`).
- `request_timeout` — таймаут HTTP-запросов в секундах (по умолчанию 10).
- `retry_attempts` — количество повторных попыток при сетевых ошибках (по умолчанию 2).

**Пример файла `config.json`**:
```json
{
  "default_lang": "en",
  "image_dir": "~/pictures/wiki",
  "request_timeout": 15,
  "retry_attempts": 3
}
```

**Загрузка конфигурации**:
- Использовать `python-dotenv` для `.env` (опционально) и стандартную библиотеку `json` для чтения конфига.
- Порядок приоритета: аргументы CLI → переменные окружения → конфигурационный файл → значения по умолчанию.

**Переменные окружения** (альтернативный способ):
- `WIKI_DEFAULT_LANG`
- `WIKI_IMAGE_DIR`
- `WIKI_TIMEOUT`
- `WIKI_RETRIES`

## 4. Логирование

**Логирование** (базовое, без внешних библиотек, можно использовать встроенный `logging`):
- При запуске с флагом `--verbose` логи выводятся в `stderr` (или в консоль).
- Уровни логирования:
  - `INFO` — основные действия: вызов API, сохранение графа, скачивание изображений.
  - `DEBUG` — детали запросов (URL, параметры, время ответа), промежуточные данные.
  - `ERROR` — ошибки API, сетевые проблемы, неверные аргументы.
- Формат лога: `[YYYY-MM-DD HH:MM:SS] [LEVEL] сообщение`.
- Логи не записываются в файл по умолчанию (только в консоль), но можно разрешить через переменную окружения `WIKI_LOG_FILE=./log.txt`.

**Реализация**:
```python
import logging

logging.basicConfig(
    level=logging.DEBUG if verbose else logging.INFO,
    format='[%(asctime)s] [%(levelname)s] %(message)s'
)
logger = logging.getLogger(__name__)
```

**Пример вывода** при `--verbose`:
```
[2026-05-24 12:34:56] [INFO] Запрос к API: https://en.wikipedia.org/w/api.php?action=query...
[2026-05-24 12:34:57] [DEBUG] Получен ответ: 200 OK, размер 1234 байт
[2026-05-24 12:34:57] [INFO] Статья 'Python' найдена
```

**Для команд с длительными операциями** (скачивание изображений, построение графа) полезно выводить прогресс-бары через `rich.progress`, но это не лог, а пользовательский интерфейс.
