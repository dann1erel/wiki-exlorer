# Функциональные требования

Разрабатываемая CLI-утилита **Wiki-Explorer** должна предоставлять пользователю возможность получать, анализировать и визуализировать информацию из Wikipedia через командную строку. Утилита должна работать с публичными API Wikipedia/Wikimedia и позволять выполнять поиск статей, получать основную информацию о страницах, анализировать ссылки, категории, изображения и статистику просмотров.

## 1. Общие функциональные требования

CLI-утилита должна поддерживать запуск из командной строки в следующем общем формате:

```bash
wiki-explorer-gpt <command> [arguments] [options]
```

Утилита должна предоставлять справку по всем доступным командам:

```bash
wiki-explorer-gpt --help
wiki-explorer-gpt <command> --help
```

Для вывода информации в консоль должны использоваться читаемые форматы: таблицы, списки, текстовые блоки, а при необходимости — графики или сохранение результатов в файл.

Утилита должна обрабатывать типовые ошибки:

```text
статья не найдена;
ошибка соединения с API;
превышение лимита запросов;
некорректные аргументы команды;
отсутствие данных по статье;
невозможность сохранить файл;
ошибка при скачивании изображения.
```

При возникновении ошибки программа не должна аварийно завершаться с техническим traceback без пояснения. Пользователь должен получить понятное сообщение о проблеме.

---

## 2. Команда `search` — поиск статей

Команда `search` должна выполнять поиск статей Wikipedia по ключевому запросу пользователя.

Пример использования:

```bash
wiki-explorer-gpt search "искусственный интеллект" --limit 20
```

Команда должна принимать обязательный аргумент:

```text
query — поисковый запрос пользователя.
```

Команда должна поддерживать опции:

```text
--limit — максимальное количество результатов;
--sort — способ сортировки результатов, например по релевантности или по дате последней правки;
--lang — язык Wikipedia, например ru или en.
```

В результате выполнения команда должна выводить список найденных статей с краткой информацией:

```text
номер результата;
название статьи;
краткий фрагмент описания;
размер страницы, если доступен;
дата последнего изменения, если доступна.
```

Команда должна использовать MediaWiki Action API с параметром `list=search`.

---

## 3. Команда `info` — базовая информация о статье

Команда `info` должна выводить основную информацию о выбранной статье.

Пример использования:

```bash
wiki-explorer-gpt info "Python" --show-categories
```

Команда должна принимать обязательный аргумент:

```text
title — название статьи.
```

Команда должна поддерживать опции:

```text
--show-categories — показать категории статьи;
--show-image — показать URL основного изображения;
--lang — язык Wikipedia.
```

Команда должна выводить:

```text
название статьи;
размер страницы в байтах;
дату последней правки;
последнего редактора;
количество внутренних ссылок, если доступно;
первые 10 категорий;
URL изображения, если оно найдено;
ссылку на статью в браузере.
```

Команда должна использовать MediaWiki Action API с параметрами:

```text
prop=info|revisions|categories|images
```

---

## 4. Команда `links` — внутренние ссылки статьи

Команда `links` должна выводить список внутренних ссылок, размещённых внутри выбранной статьи.

Пример использования:

```bash
wiki-explorer-gpt links "Python" --search "programming" --limit 50
```

Команда должна принимать обязательный аргумент:

```text
title — название статьи.
```

Команда должна поддерживать опции:

```text
--limit — максимальное количество ссылок;
--search — фильтрация ссылок по части названия;
--lang — язык Wikipedia.
```

Команда должна выводить список страниц, на которые ссылается выбранная статья.

Если пользователь указал параметр `--search`, утилита должна отобразить только те ссылки, в названии которых есть указанная строка.

Команда должна использовать MediaWiki Action API с параметром:

```text
prop=links
```

---

## 5. Команда `graph` — построение графа ссылок

Команда `graph` должна строить простой граф связей между исходной статьёй и страницами, на которые она ссылается.

Пример использования:

```bash
wiki-explorer-gpt graph "Python" --output graph.png --format png
```

Команда должна принимать обязательный аргумент:

```text
title — название исходной статьи.
```

Команда должна поддерживать опции:

```text
--limit — максимальное количество ссылок для построения графа;
--output — путь для сохранения файла с графом;
--format — формат сохранения, например png или pdf;
--text — вывести граф в текстовом виде без изображения;
--depth - глубина графа;
--lang — язык Wikipedia.
```

В графе:

```text
узлами должны быть статьи;
центральным узлом должна быть исходная статья;
рёбра должны показывать направление ссылки от исходной статьи к связанной статье.
```

Если графическая визуализация недоступна, команда должна уметь выводить список рёбер в текстовом формате:

```text
Python -> Programming language
Python -> Guido van Rossum
Python -> Software
```

Команда должна использовать MediaWiki Action API с параметром:

```text
prop=links
```

Для построения графа могут использоваться библиотеки `networkx` и `matplotlib`.

---

## 6. Команда `categories` — категории статьи

Команда `categories` должна выводить категории, к которым относится выбранная статья.

Пример использования:

```bash
wiki-explorer-gpt categories "Python" --tree
```

Команда должна принимать обязательный аргумент:

```text
title — название статьи.
```

Команда должна поддерживать опции:

```text
--tree — вывести категории в виде простого дерева;
--limit — максимальное количество категорий;
--lang — язык Wikipedia.
```

В обычном режиме команда должна выводить плоский список категорий.

В режиме `--tree` команда должна дополнительно попытаться получить категории верхнего уровня или подкатегории первого уровня, если это возможно через API.

Команда должна использовать MediaWiki Action API с параметрами:

```text
prop=categories
list=categorymembers
```

---

## 7. Команда `pageviews` — статистика просмотров статьи

Команда `pageviews` должна получать статистику просмотров выбранной статьи за указанный период.

Пример использования:

```bash
wiki-explorer-gpt pageviews "Python" --days 90 --chart ascii
```

Команда должна принимать обязательный аргумент:

```text
title — название статьи.
```

Команда должна поддерживать опции:

```text
--days — количество дней для анализа, по умолчанию 30;
--chart — тип графика: none, ascii или png;
--output — путь для сохранения графика;
```

Команда должна выводить:

```text
таблицу с датами и количеством просмотров;
суммарное количество просмотров за период;
среднее количество просмотров в день;
день с максимальным количеством просмотров;
день с минимальным количеством просмотров.
```

Если выбран режим `--chart ascii`, программа должна вывести простой текстовый график в консоль.

Если выбран режим `--chart png`, программа должна сохранить график в файл.

Команда должна использовать Wikimedia Pageviews API.

---

## 8. Команда `random` — случайная статья

Команда `random` должна возвращать случайную статью Wikipedia.

Пример использования:

```bash
wiki-explorer-gpt random --category "Наука" --min-words 1000 --with-image
```

Команда должна поддерживать опции:

```text
--category — категория, из которой нужно получить случайную статью;
--min-words — минимальный примерный размер статьи в словах;
--with-image — искать статью, у которой есть изображение;
--lang — язык Wikipedia.
```

Если категория не указана, команда должна возвращать случайную статью из основного пространства имён Wikipedia.

Команда должна выводить:

```text
название статьи;
краткую информацию о статье;
ссылку на статью;
размер статьи;
информацию о наличии изображения.
```

Команда должна использовать MediaWiki Action API с параметром:

```text
list=random
```

Дополнительно для проверки размера и изображений могут использоваться:

```text
prop=info|images
```

---

## 9. Команда `images` — изображения статьи

Команда `images` должна выводить список изображений, используемых в выбранной статье, и при необходимости скачивать их.

Пример использования:

```bash
wiki-explorer-gpt images "Солнечная система" --download --output ./pictures
```

Команда должна принимать обязательный аргумент:

```text
title — название статьи.
```

Команда должна поддерживать опции:

```text
--download — скачать изображения;
--output — папка для сохранения изображений;
--limit — максимальное количество изображений;
--all — скачать все найденные изображения;
--lang — язык Wikipedia.
```

Команда должна выводить:

```text
номер изображения;
название файла;
URL изображения или превью;
размер файла, если доступен;
тип файла, если доступен.
```

Если пользователь указал `--download`, программа должна скачать изображения в указанную папку.

Если папка не существует, программа должна создать её или вывести понятную ошибку.

Команда должна использовать MediaWiki Action API с параметрами:

```text
prop=images
prop=imageinfo
```

---

## 11. Требования к работе с REST API

Утилита должна обязательно использовать публичные REST/HTTP API Wikipedia/Wikimedia.

Основные API:

```text
MediaWiki Action API
Wikimedia Pageviews API
```

Каждая основная команда должна выполнять HTTP-запрос к API и обрабатывать JSON-ответ.

Программа должна корректно обрабатывать:

```text
пустой ответ API;
отсутствие статьи;
ошибочный HTTP-статус;
проблемы с сетью;
ограничение количества запросов;
некорректный JSON-ответ.
```

---

## 12. Требования к пользовательскому выводу

Информация должна выводиться в удобном для пользователя виде.

Для табличных данных желательно использовать таблицы.

Например, для команды `search`:

```text
№ | Название | Описание | Размер
```

Для команды `pageviews`:

```text
Дата | Просмотры
```

Для команды `links`:

```text
№ | Ссылка
```

Сообщения об ошибках должны быть понятными:

```text
Статья "..." не найдена.
Не удалось подключиться к Wikipedia API.
Не удалось сохранить файл по указанному пути.
По заданному запросу ничего не найдено.
```

# Технические требования

## 1. Используемые библиотеки

Проект должен быть реализован на языке программирования **Python**. Рекомендуемая версия Python:

```text
Python 3.10+
```

Для разработки CLI-утилиты должны использоваться следующие библиотеки.

### Основные библиотеки

```text
click
```

Библиотека для создания интерфейса командной строки. С её помощью должны быть реализованы основные команды и подкоманды утилиты: `search`, `info`, `links`, `graph`, `categories`, `pageviews`, `random`, `images`.

```text
requests
```

Библиотека для выполнения HTTP-запросов к внешним API Wikipedia/Wikimedia.

```text
rich
```

Библиотека для красивого вывода информации в терминал. Должна использоваться для отображения таблиц, сообщений об ошибках, заголовков и форматированного текста.

### Библиотеки для обработки данных

```text
pandas
```

Может использоваться для обработки табличных данных, например при работе со статистикой просмотров статьи или при экспорте результатов в CSV.

```text
json
csv
```

Стандартные библиотеки Python для экспорта данных в файлы форматов JSON и CSV.

### Библиотеки для построения графов и визуализации

```text
networkx
```

Библиотека для построения графа связей между статьями.

```text
matplotlib
```

Библиотека для визуализации графа ссылок и построения графиков просмотров статьи.

### Библиотеки для работы с файлами и изображениями

```text
pathlib
```

Стандартная библиотека Python для удобной работы с путями к файлам и папкам.

```text
Pillow
```

Может использоваться для дополнительной обработки изображений, если в проекте будет реализована загрузка или проверка изображений.

### Библиотеки для тестирования

```text
pytest
```

Библиотека для написания и запуска unit-тестов.

```text
requests-mock
```

Библиотека для тестирования функций, которые выполняют HTTP-запросы к API, без реального обращения к Wikipedia.

## 3. Обработка ошибок

CLI-утилита должна корректно обрабатывать ошибки и выводить пользователю понятные сообщения без необработанных traceback-сообщений.

### Ошибки пользовательского ввода

Программа должна проверять корректность аргументов команд.

Необходимо обрабатывать следующие ситуации:

```text
не указано название статьи;
не указан поисковый запрос;
передан некорректный лимит результатов;
передано отрицательное количество дней для статистики;
указан неподдерживаемый формат экспорта;
указан некорректный путь для сохранения файла.
```

Пример сообщения:

```text
Ошибка: параметр --limit должен быть положительным числом.
```

### Ошибки при работе с API

Программа должна обрабатывать ошибки, связанные с HTTP-запросами:

```text
нет подключения к интернету;
API Wikipedia временно недоступен;
превышено время ожидания ответа;
получен HTTP-статус 404, 429 или 500;
ответ API не является корректным JSON;
API вернул пустой результат.
```

Пример сообщения:


## 4. REST API endpoints

CLI-утилита должна использовать публичные HTTP API Wikipedia/Wikimedia. Основными API являются:

```text
MediaWiki Action API
Wikimedia Pageviews API
```

Базовый endpoint для MediaWiki Action API:

```text
https://{lang}.wikipedia.org/w/api.php
```

Где `{lang}` — код языка Wikipedia, например:

```text
ru
en
de
fr
```

Базовый endpoint для Wikimedia Pageviews API:

```text
https://wikimedia.org/api/rest_v1/metrics/pageviews/per-article/{project}/{access}/{agent}/{article}/{granularity}/{start}/{end}
```

Где:

```text
project — проект Wikipedia, например ru.wikipedia.org или en.wikipedia.org;
access — способ доступа, например all-access;
agent — тип пользователя, например user;
article — название статьи;
granularity — детализация статистики, например daily;
start — дата начала периода;
end — дата конца периода.
```

---

```text
Поиск статей по ключевому запросу.
```

```text
action=query

srsearch={query}
```


API:

```text

```

```text

```

Пример использования:

```text

```

Команда должна сначала получить список изображений, затем получить URL каждого изображения через `imageinfo`, после чего при необходимости скачать изображения в указанную папку.
